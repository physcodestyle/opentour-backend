## Соглашение о именах
Если вы дополняете код или документацию проекта, пожалуйста, соблюдайте принятые соглашения о именовании в целях поддержания однообразия документации :)

### Спецификация OpenAPI

- Для схем данных (объекты `schemes`) - **CamelCase**
- Для атрибутов схем данных (`properties`) и параметров запроса - **lowerCamelCase**
- Для параметров запроса, передаваемых в пути ресурса (`in:path`) - **snake_case**
- Для путей ресурсов `paths` - **kebab-case**

Также не забывайте оставлять человекопонятные описания объектов и эндпоинтов - `summary` и `description`.
